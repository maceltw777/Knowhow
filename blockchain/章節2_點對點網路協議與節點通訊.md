## 2.1 P2P 協議棧

### TCP/IP 底層連接

比特幣等區塊鏈網路建立在 TCP/IP 協議棧之上，透過 TCP 長連接確保可靠的資料傳輸。

#### 連接參數
- **端口號**：Bitcoin 主網 8333，測試網 18333
- **連接池大小**：8 個出站連接，最多 125 個入站連接
- **TCP_NODELAY**：禁用 Nagle 演算法，降低延遲
- **連接超時**：通常設置為 60 秒

#### 節點狀態機
```
DISCONNECTED → CONNECTING → CONNECTED → HANDSHAKING → READY → DISCONNECTED
```

#### Python Socket 實現示例

```python
import socket
import struct
import hashlib

class BitcoinNode:
    def __init__(self, host, port=8333):
        self.host = host
        self.port = port
        self.socket = None
        self.version = 70015
        
    def connect(self):
        self.socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        self.socket.setsockopt(socket.IPPROTO_TCP, socket.TCP_NODELAY, 1)
        self.socket.settimeout(60)
        self.socket.connect((self.host, self.port))
        
    def send_message(self, command, payload=b''):
        magic = b'\xf9\xbe\xb4\xd9'  # 主網 Magic
        cmd = command.encode('ascii').ljust(12, b'\x00')
        length = struct.pack('<I', len(payload))
        checksum = hashlib.sha256(hashlib.sha256(payload).digest()).digest()[:4]
        header = magic + cmd + length + checksum
        self.socket.sendall(header + payload)
```

#### DDoS 防護機制
- **IP 封鎖**：對惡意行為的 IP 進行封鎖
- **連接限制**：每個 IP 最多 1 個入站連接
- **速率限制**：限制消息發送頻率
- **行為評分**：累計惡意行為後降分並最終封鎖

### 消息格式設計

比特幣網路訊息格式是一套精心設計的二進制協議，確保節點間通訊的高效性和可靠性。

#### 24 字節訊息頭部格式

```
[Magic 4B][Command 12B][Length 4B][Checksum 4B][Payload ...]
```

| 欄位 | 大小 | 說明 |
|------|------|------|
| Magic | 4 bytes | 網路識別碼，主網: 0xF9BEB4D9 |
| Command | 12 bytes | ASCII 命令名，不足補 0x00 |
| Length | 4 bytes | Payload 長度，小端序 |
| Checksum | 4 bytes | SHA256(SHA256(Payload)) 前4字節 |
| Payload | 可變 | 具體訊息內容 |

#### 重要訊息類型

| 命令 | 用途 |
|------|------|
| version | 握手，交換版本資訊 |
| verack | 確認版本資訊 |
| inv | 通知對方有新交易/區塊 |
| getdata | 請求特定交易/區塊資料 |
| block | 傳送完整區塊資料 |
| tx | 傳送交易資料 |
| ping/pong | 連接存活檢測 |
| addr | 分享已知節點地址列表 |

#### Python 序列化/反序列化實現

```python
import struct
import hashlib

def create_message(magic, command, payload):
    """創建比特幣網路訊息"""
    cmd_bytes = command.encode('ascii').ljust(12, b'\x00')[:12]
    length = struct.pack('<I', len(payload))
    checksum = hashlib.sha256(hashlib.sha256(payload).digest()).digest()[:4]
    return magic + cmd_bytes + length + checksum + payload

def parse_message_header(data):
    """解析訊息頭部"""
    if len(data) < 24:
        raise ValueError("頭部資料不足")
    magic = data[0:4]
    command = data[4:16].rstrip(b'\x00').decode('ascii')
    length = struct.unpack('<I', data[16:20])[0]
    checksum = data[20:24]
    return magic, command, length, checksum

def verify_checksum(payload, expected_checksum):
    """驗證訊息校驗和"""
    actual = hashlib.sha256(hashlib.sha256(payload).digest()).digest()[:4]
    return actual == expected_checksum

# 構建 version 訊息
def build_version_message(version, services, timestamp, addr_recv, addr_from, nonce, user_agent, start_height):
    payload = struct.pack('<i', version)
    payload += struct.pack('<Q', services)
    payload += struct.pack('<q', timestamp)
    # addr_recv, addr_from, nonce, user_agent, start_height...
    return payload

BITCOIN_MAINNET_MAGIC = b'\xf9\xbe\xb4\xd9'
BITCOIN_TESTNET_MAGIC = b'\x0b\x11\x09\x07'
```

#### 設計考量
- **Magic Number**：防止不同網路間的消息混淆
- **Checksum**：確保傳輸過程中的資料完整性
- **Version 談判**：握手時協商雙方支援的版本
- **訊息大小限制**：防止 OOM 攻擊，最大 32MB

## 2.2 節點發現與 DHT

### Kademlia 演算法

Kademlia 是一種分散式雜湊表 (DHT) 演算法，被用於比特幣等 P2P 網路的節點發現。

#### 核心概念

**160-bit 節點 ID**
- 每個節點隨機分配 160 位 ID（通常是公鑰哈希）
- ID 空間均勻分布，確保負載均衡

**XOR 距離度量**
```python
def distance(id1, id2):
    return id1 ^ id2  # 按位異或
```
- 對稱性：`d(A, B) == d(B, A)`
- 三角不等式：`d(A, C) <= d(A, B) XOR d(B, C)`

**k-桶 (k-bucket) 路由表**
- 按距離遠近分成 160 個桶，每桶最多 k=20 個節點
- 近距離桶存儲更多已知節點（偏好近鄰）
- LRU 淘汰策略：優先保留長時間在線的節點

#### FIND_NODE 演算法
```
1. 從自己的路由表選擇最近的 α=3 個節點
2. 並行向這些節點發送 FIND_NODE(target) 請求
3. 收集返回的更近節點，更新候選列表
4. 對新發現的更近節點繼續查詢
5. 直到找到目標節點或路由表穩定
```

時間複雜度：O(log N) 次網路跳躍

#### Python 實現

```python
import hashlib
import random
from dataclasses import dataclass

@dataclass
class NodeInfo:
    node_id: int
    ip: str
    port: int
    last_seen: float = 0

class KBucket:
    def __init__(self, k=20):
        self.k = k
        self.nodes = []  # 按最近活躍時間排序
        
    def add_node(self, node: NodeInfo):
        # 檢查是否已存在
        for i, n in enumerate(self.nodes):
            if n.node_id == node.node_id:
                # 移到尾部（最近見到）
                self.nodes.pop(i)
                self.nodes.append(node)
                return
        
        if len(self.nodes) < self.k:
            self.nodes.append(node)
        else:
            # 桶已滿，檢查最舊的節點是否仍在線
            oldest = self.nodes[0]
            if not self.ping(oldest):
                self.nodes.pop(0)
                self.nodes.append(node)
            # 若最舊節點仍在線，丟棄新節點（偏好老節點）
    
    def get_closest(self, target_id, count=20):
        return sorted(self.nodes, 
                      key=lambda n: n.node_id ^ target_id)[:count]

class RoutingTable:
    def __init__(self, local_id, k=20):
        self.local_id = local_id
        self.k = k
        self.buckets = [KBucket(k) for _ in range(160)]
        
    def bucket_index(self, node_id):
        distance = self.local_id ^ node_id
        if distance == 0:
            return 0
        return distance.bit_length() - 1
    
    def add_node(self, node: NodeInfo):
        idx = self.bucket_index(node.node_id)
        self.buckets[idx].add_node(node)
    
    def find_closest(self, target_id, count=20):
        result = []
        for bucket in self.buckets:
            result.extend(bucket.get_closest(target_id, count))
        return sorted(result, key=lambda n: n.node_id ^ target_id)[:count]

class KademliaNode:
    def __init__(self):
        self.node_id = random.getrandbits(160)
        self.routing_table = RoutingTable(self.node_id)
        self.alpha = 3  # 並行查詢數
    
    def find_node(self, target_id):
        """查找最近的節點"""
        closest = self.routing_table.find_closest(target_id, self.alpha)
        queried = set()
        
        while True:
            to_query = [n for n in closest if n.node_id not in queried][:self.alpha]
            if not to_query:
                break
                
            for node in to_query:
                queried.add(node.node_id)
                # 向 node 發送 FIND_NODE 請求
                response = self.send_find_node(node, target_id)
                for new_node in response:
                    self.routing_table.add_node(new_node)
            
            new_closest = self.routing_table.find_closest(target_id, 20)
            if new_closest == closest:
                break
            closest = new_closest
        
        return closest
```

#### 路由表維護
- **定期刷新**：每小時刷新空桶，通過隨機查詢填充
- **節點驗證**：定期 ping 桶中的節點，確保其仍在線
- **加入網路**：加入時進行自我查詢，填充路由表

## 2.3 Gossip 協議

### Gossip 流言傳播

Gossip 協議（又稱 Epidemic Protocol）是區塊鏈網路中用於廣播交易和區塊的核心協議。

#### 傳播模型

**隨機扇出傳播**
1. 發起節點選擇 `fanout` 個隨機鄰居
2. 每個收到訊息的節點再選擇 `fanout` 個鄰居繼續傳播
3. 已接收過訊息的節點不再轉發（去重機制）

**傳播複雜度**
- 輪數：O(log N)（N 為網路節點數）
- 訊息副本數：O(N log N)
- 單節點負擔：O(fanout) 每輪

#### 交易廣播實現

```python
import hashlib
import time
from typing import Set, Dict, List, Optional

class TransactionGossip:
    def __init__(self, node_id: str, fanout: int = 8):
        self.node_id = node_id
        self.fanout = fanout
        self.seen_txids: Set[str] = set()  # 已見過的交易哈希
        self.peers: List[str] = []  # 已知鄰居節點
        
    def receive_transaction(self, tx_data: bytes, source: str) -> bool:
        """接收並轉發交易"""
        tx_hash = hashlib.sha256(tx_data).hexdigest()
        
        if tx_hash in self.seen_txids:
            return False  # 已收過，不再轉發
        
        self.seen_txids.add(tx_hash)
        self.gossip_transaction(tx_data, exclude=source)
        return True
    
    def gossip_transaction(self, tx_data: bytes, exclude: Optional[str] = None):
        """隨機選擇節點轉發"""
        eligible_peers = [p for p in self.peers if p != exclude]
        selected = random.sample(eligible_peers, min(self.fanout, len(eligible_peers)))
        
        for peer in selected:
            self.send_to_peer(peer, tx_data)

class BlockGossip:
    def __init__(self, node_id: str, fanout: int = 16):
        self.node_id = node_id
        self.fanout = fanout
        self.seen_blocks: Dict[str, float] = {}  # hash -> timestamp
        self.peers: List[str] = []
        
    def receive_block(self, block_data: bytes, source: str) -> bool:
        block_hash = hashlib.sha256(block_data).hexdigest()
        
        if block_hash in self.seen_blocks:
            return False
        
        # 驗證區塊
        if not self.validate_block(block_data):
            return False
        
        self.seen_blocks[block_hash] = time.time()
        self.gossip_block(block_data, exclude=source)
        return True
    
    def gossip_block(self, block_data: bytes, exclude: Optional[str] = None):
        eligible_peers = [p for p in self.peers if p != exclude]
        selected = random.sample(eligible_peers, min(self.fanout, len(eligible_peers)))
        
        for peer in selected:
            self.send_to_peer(peer, block_data)

class MessageDeduplication:
    """訊息去重機制"""
    def __init__(self, ttl: int = 3600):
        self.seen_messages: Dict[str, float] = {}
        self.ttl = ttl  # 訊息保留時間（秒）
        
    def is_duplicate(self, msg_hash: str) -> bool:
        now = time.time()
        
        # 清理過期記錄
        expired = [h for h, t in self.seen_messages.items() if now - t > self.ttl]
        for h in expired:
            del self.seen_messages[h]
        
        if msg_hash in self.seen_messages:
            return True
        
        self.seen_messages[msg_hash] = now
        return False
```

#### 洪泛防護機制

```python
class RateLimiter:
    def __init__(self, max_per_second: int = 100):
        self.max_per_second = max_per_second
        self.peer_counts: Dict[str, List[float]] = {}
        
    def should_accept(self, peer_id: str) -> bool:
        now = time.time()
        if peer_id not in self.peer_counts:
            self.peer_counts[peer_id] = []
        
        # 清理 1 秒前的記錄
        self.peer_counts[peer_id] = [
            t for t in self.peer_counts[peer_id] if now - t < 1.0
        ]
        
        if len(self.peer_counts[peer_id]) >= self.max_per_second:
            return False
        
        self.peer_counts[peer_id].append(now)
        return True
```

#### Gossip 協議優化
- **inv-getdata 兩階段**：先廣播交易哈希（inv），按需請求完整資料（getdata）
- **Erlay 協議**：BIP330，利用集合調和 (set reconciliation) 大幅降低頻寬
- **Dandelion++**：增強隱私，先在莖（stem）階段單路傳播，再切換到普通 gossip

> **🔄 技術更新（2025+）**：
> - **libp2p**：Ethereum 自 The Merge 後已全面採用 libp2p 作為 P2P 層框架（取代自研協議），整合 noise 加密握手、yamux 多路複用、mDNS 本地節點發現等模組，是現代區塊鏈 P2P 開發的主流方案，Polkadot、Filecoin 等也採用。
> - **Erlay（BIP330）**：截至 2025 年仍在 Bitcoin Core 的開發整合中，預期可將 P2P 頻寬需求降低約 40%。
