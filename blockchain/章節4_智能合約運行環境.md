## 4.1 EVM 指令集架構

### EVM 操作碼與執行模型

以太坊虛擬機 (EVM) 是一個基於棧的虛擬機，執行 Solidity 等高級語言編譯後的字節碼。

#### 核心設計特性

| 特性 | 規格 |
|------|------|
| 機器類型 | 棧式虛擬機 (Stack Machine) |
| 棧最大深度 | 1024 |
| 字長 | 256-bit (32 bytes) |
| 存儲單元 | 256-bit slots |
| 內存模型 | 線性字節陣列（按需擴展） |

#### EVMStack 實現

```python
class EVMStack:
    def __init__(self):
        self.stack = []
        self.max_depth = 1024
    
    def push(self, value: int):
        if len(self.stack) >= self.max_depth:
            raise StackOverflowError("Stack overflow (max depth 1024)")
        self.stack.append(value & ((1 << 256) - 1))  # 截斷到 256 位
    
    def pop(self) -> int:
        if not self.stack:
            raise StackUnderflowError("Stack underflow")
        return self.stack.pop()
    
    def peek(self, n=0) -> int:
        if n >= len(self.stack):
            raise IndexError(f"Stack peek at {n} out of range")
        return self.stack[-(n+1)]
    
    def swap(self, n: int):
        """SWAP1 ~ SWAP16: 交換棧頂與第 n 個元素"""
        if n >= len(self.stack):
            raise InvalidOperationError(f"SWAP{n} requires {n+1} items")
        self.stack[-1], self.stack[-(n+1)] = self.stack[-(n+1)], self.stack[-1]
    
    def dup(self, n: int):
        """DUP1 ~ DUP16: 複製第 n 個元素到棧頂"""
        if n > len(self.stack):
            raise InvalidOperationError(f"DUP{n} requires {n} items")
        self.stack.append(self.stack[-(n)])
```

#### EVMMemory 實現

```python
class EVMMemory:
    def __init__(self):
        self.memory = bytearray()
    
    def _extend_to(self, offset: int, size: int):
        """按需擴展內存，以 32 字節對齊"""
        required = offset + size
        if required > len(self.memory):
            # 向上對齊到 32 字節
            new_size = ((required + 31) // 32) * 32
            self.memory.extend(b'\x00' * (new_size - len(self.memory)))
    
    def store(self, offset: int, value: int, size: int = 32):
        self._extend_to(offset, size)
        value_bytes = value.to_bytes(size, 'big')
        self.memory[offset:offset+size] = value_bytes
    
    def load(self, offset: int, size: int = 32) -> int:
        self._extend_to(offset, size)
        return int.from_bytes(self.memory[offset:offset+size], 'big')
    
    def size(self) -> int:
        return len(self.memory)
```

```

  CHAINID      (0x46) → 2 gas  鏈 ID

  JUMP  (0x56) → 8 gas    無條件跳轉
  JUMPI (0x57) → 10 gas   條件跳轉

  STOP  (0x00) → 0 gas    停止執行

  REVERT (0xFD) → 0 gas   回滾並返回數據

調用操作碼：
  CALL         (0xF1) → 700 base gas   調用另一合約
  DELEGATECALL (0xF4) → 700 base gas   代理調用（保留 caller/value）
  STATICCALL   (0xFA) → 700 base gas   靜態調用（不可修改狀態）
  CREATE       (0xF0) → 32000 gas      創建合約
  CREATE2      (0xF5) → 32000 gas      確定性地址創建合約
```

## 4.2 Solidity 編譯器與 ABI

### Solidity 編譯優化與 ABI

Solidity 編譯器將高級合約代碼轉換為 EVM 字節碼，並生成 ABI（Application Binary Interface）。

#### 編譯管線

```
Solidity 源碼
    ↓
[1] 詞法分析 (Lexical Analysis)  → Token 流
    ↓
[2] 語法分析 (Syntax Analysis)   → AST（抽象語法樹）
    ↓
[3] 語義分析 (Semantic Analysis) → 類型檢查、作用域解析
    ↓
[4] 代碼生成 (Code Generation)   → EVM 字節碼
    ↓
[5] ABI 生成                     → JSON 介面描述
```

#### SolidityCompiler 實現

```python
class SolidityCompiler:
    def __init__(self):
        self.lexer = Lexer()
        self.parser = Parser()
        self.analyzer = SemanticAnalyzer()
        self.generator = BytecodeGenerator()
    
    def compile(self, source_code: str):
        tokens = self.lexer.tokenize(source_code)
        ast = self.parser.parse(tokens)
        analyzed_ast = self.analyzer.analyze(ast)
        bytecode = self.generator.generate(analyzed_ast)
        abi = self.generate_abi(analyzed_ast)
        return {'bytecode': bytecode, 'abi': abi}
    
    def generate_abi(self, ast) -> list:
        abi = []
        for node in ast.get_functions():
            abi_entry = {
                'type': 'function',
                'name': node.name,
                'inputs': [{'name': p.name, 'type': p.type} for p in node.parameters],
                'outputs': [{'type': r.type} for r in node.returns],
                'stateMutability': node.mutability  # 'pure', 'view', 'nonpayable', 'payable'
            }
            abi.append(abi_entry)
        return abi
```

#### 函數分派器

```python
class BytecodeGenerator:
    def generate_function_dispatcher(self, functions):
        """生成函數選擇器分派代碼"""
        bytecode = []
        
        # CALLDATASIZE check
        bytecode.extend([0x36, 0x60, 0x04, 0x10])  # CALLDATASIZE < 4 → fallback
        
        for func in functions:
            selector = self.get_function_selector(func)
            # 比較 calldata 前 4 bytes 和選擇器
            bytecode.extend([
                0x60, 0x00,  # PUSH1 0x00
                0x35,        # CALLDATALOAD
                0x60, 0xE0,  # PUSH1 224
                0x1C,        # SHR（取前4字節）
                0x63, *selector.to_bytes(4, 'big'),  # PUSH4 <selector>
                0x14,        # EQ
                0x61, *func.offset.to_bytes(2, 'big'),  # PUSH2 <offset>
                0x57,        # JUMPI
            ])
        
        return bytes(bytecode)
    
    def get_function_selector(self, func) -> int:
        """計算 keccak256(函數簽名) 前 4 字節"""
        signature = f"{func.name}({','.join(p.type for p in func.parameters)})"
        return int.from_bytes(keccak256(signature.encode())[:4], 'big')
```

#### Solidity 關鍵字

```
可見性修飾符：
  public    → 外部和內部都可訪問，自動生成 getter
  private   → 僅當前合約可訪問
  internal  → 當前合約及繼承合約可訪問
  external  → 僅外部調用（gas 更省，因為使用 calldata）

狀態可變性：
  pure      → 不讀寫狀態
  view      → 只讀狀態，不修改
  payable   → 可接收 ETH
  (無修飾)  → 可讀寫狀態

特殊函數：
  constructor → 部署時執行一次
  receive     → 接收純 ETH 轉帳
  fallback    → 沒有匹配函數時調用
```

#### ABI 編碼規範

```
函數調用 = 4字節選擇器 + ABI 編碼的參數

ABI 類型編碼：
  uint256     → 32 字節，左對齊（補零）
  address     → 32 字節（20字節地址 + 12字節零填充）
  bool        → 32 字節（0 或 1）
  bytes       → 偏移量 + 長度 + 數據（32字節對齊）
  string      → 同 bytes
  uint256[]   → 偏移量 + 長度 + 每個元素32字節

示例：transfer(address to, uint256 amount)
選擇器 = keccak256("transfer(address,uint256)") = 0xa9059cbb
編碼  = 0xa9059cbb
        000000000000000000000000[to_address_20bytes]
        [amount_32bytes]
```

#### 優化選項

| 優化選項 | 說明 | Gas 節省 |
|---------|------|---------|
| `--optimize` | 啟用優化器 | 10-40% |
| `--optimize-runs` | 優化目標執行次數 (默認200) | 部署 vs 運行成本均衡 |

## 4.3 Gas 模型

### Gas 經濟學與計量模型

Gas 是以太坊計算資源的計量單位，防止無限循環和資源濫用，同時激勵驗證者。

#### 交易 Gas 計算

```python
def calculate_transaction_gas(tx) -> int:
    gas = 21000  # 基礎交易成本
    
    # 輸入數據成本
    for byte in tx.data:
        if byte == 0:
            gas += 4   # 零字節成本低
        else:
            gas += 16  # 非零字節成本高（EIP-2028後從68降至16）
    
    # 合約創建
    if tx.to is None:
        gas += 32000  # CREATE 基礎成本
    
    # 訪問列表 (EIP-2930)
    for address in tx.access_list:
        gas += 2400  # 每個地址
        for storage_key in tx.access_list[address]:
            gas += 1900  # 每個存儲鍵
    
    return gas
```

#### 內存擴展成本

```python
def memory_expansion_cost(current_size: int, new_size: int) -> int:
    """計算內存擴展的 Gas 成本"""
    # 向上對齊到 32 字節
    old_words = (current_size + 31) // 32
    new_words = (new_size + 31) // 32
    
    if new_words <= old_words:
        return 0
    
    def memory_cost(words):
        return 3 * words + words * words // 512
    
    return memory_cost(new_words) - memory_cost(old_words)
```

#### EIP-1559 Gas 改革

```python
class EIP1559Transaction:
    def __init__(self, max_fee_per_gas, max_priority_fee_per_gas, gas_limit):
        self.max_fee = max_fee_per_gas             # 用戶願意支付的最高 Gas 價格
        self.max_priority_fee = max_priority_fee_per_gas  # 給驗證者的小費
        self.gas_limit = gas_limit

def calculate_effective_gas_price(tx, base_fee):
    """計算實際 Gas 價格"""
    priority_fee = min(tx.max_priority_fee, tx.max_fee - base_fee)
    return base_fee + priority_fee

def update_base_fee(parent_gas_used, parent_gas_limit, parent_base_fee):
    """動態調整 Base Fee"""
    target = parent_gas_limit // 2  # 目標 50% 使用率
    
    if parent_gas_used > target:
        # 區塊超過半滿 → 提高 base fee
        delta = parent_base_fee * (parent_gas_used - target) // target // 8
        return parent_base_fee + max(delta, 1)
    else:
        # 區塊低於半滿 → 降低 base fee
        delta = parent_base_fee * (target - parent_gas_used) // target // 8
        return parent_base_fee - delta
```

#### Gas 估算

```python
def estimate_gas(transaction, state, binary_search=True) -> int:
    """二分查找最小 Gas 限制"""
    if not binary_search:
        # 簡單方法：直接模擬
        return simulate_transaction(transaction, state, gas_limit=30_000_000)
    
    low, high = 21000, 30_000_000  # 最低基礎成本 to 區塊 Gas 上限
    
    while low + 1 < high:
        mid = (low + high) // 2
        try:
            simulate_transaction(transaction, state, gas_limit=mid)
            high = mid  # 成功，嘗試更少
        except OutOfGasError:
            low = mid   # 失敗，需要更多
    
    return high
```

#### Gas 優化技巧

| 技巧 | Gas 節省 | 原理 |
|------|---------|------|
| 使用 `calldata` 而非 `memory` | 高 | calldata 不可修改，成本低 |
| 打包 Struct（256位對齊） | 中 | 減少 SLOAD/SSTORE 次數 |
| 使用 `immutable` 變量 | 高 | 避免 SLOAD，直接 PUSH |
| 短路評估 `&&` `||` | 低 | 避免不必要的計算 |
| 批量操作 | 中 | 分攤 21000 基礎成本 |
| 避免使用 `delete` 之外清零 | 中 | 利用 SSTORE 返還機制 |
| unchecked 算術 | 低-中 | Solidity 0.8+ 默認溢出檢查 |

#### Gas 重入攻擊防護

```solidity
// 不安全：先轉帳後更新狀態
function withdraw(uint amount) external {
    require(balances[msg.sender] >= amount);
    (bool success,) = msg.sender.call{value: amount}("");  // 可重入！
    balances[msg.sender] -= amount;
}

// 安全：CEI 模式（Checks-Effects-Interactions）
function withdraw(uint amount) external {
    require(balances[msg.sender] >= amount);    // Check
    balances[msg.sender] -= amount;              // Effect（先更新）
    (bool success,) = msg.sender.call{value: amount}("");  // Interaction
    require(success);
}
```
