## 9.1 跨鏈橋接技術

### 跨鏈橋接協議實現

跨鏈橋允許資產和信息在不同區塊鏈之間傳遞，是 DeFi 多鏈生態的基礎設施。

#### 橋接類型

```python
from enum import Enum
from dataclasses import dataclass, field
from typing import Dict, List, Optional
import hashlib
import time

class BridgeType(Enum):
    LOCK_MINT = "lock_mint"       # 鎖定-鑄造（最常見）
    BURN_MINT = "burn_mint"       # 銷毀-鑄造（原生資產）
    LIQUIDITY_POOL = "liq_pool"   # 流動性池（無需發行包裝代幣）
    NATIVE_SWAP = "native"        # 原子交換

class TransactionStatus(Enum):
    PENDING = "pending"
    LOCKED = "locked"             # 源鏈已鎖定
    RELAYED = "relayed"           # 中繼節點已確認
    MINTED = "minted"             # 目標鏈已鑄造
    COMPLETED = "completed"
    FAILED = "failed"
```

#### CrossChainTransaction 數據結構

```python
@dataclass
class CrossChainTransaction:
    tx_id: str
    bridge_type: BridgeType
    source_chain: str              # 源鏈 ID（e.g., "ethereum"）
    target_chain: str              # 目標鏈 ID（e.g., "polygon"）
    source_token: str              # 源鏈代幣地址
    target_token: str              # 目標鏈代幣地址（或包裝代幣）
    sender: str                    # 源鏈發送者
    recipient: str                 # 目標鏈接收者
    amount: int                    # 轉帳金額（最小單位）
    source_tx_hash: str = ''       # 源鏈交易哈希
    target_tx_hash: str = ''       # 目標鏈交易哈希
    status: TransactionStatus = TransactionStatus.PENDING
    validator_signatures: List[bytes] = field(default_factory=list)
    merkle_proof: Optional[dict] = None
    created_at: int = field(default_factory=lambda: int(time.time()))
```

#### CrossChainBridge 主類

```python
class CrossChainBridge:
    def __init__(self, bridge_type: BridgeType):
        self.bridge_type = bridge_type
        self.supported_chains: Dict[str, dict] = {}   # chain_id → config
        self.validators: List[str] = []               # 驗證者地址列表
        self.threshold: int = 0                       # 最少需要的驗證者簽名數
        self.pending_txs: Dict[str, CrossChainTransaction] = {}
        self.nonce_tracker: Dict[str, int] = {}       # 防止重放攻擊
    
    def register_chain(self, chain_id: str, config: dict):
        """注冊支持的區塊鏈"""
        required_fields = ['rpc_endpoint', 'bridge_contract', 'native_token']
        for field in required_fields:
            if field not in config:
                raise ValueError(f"缺少必要配置：{field}")
        
        self.supported_chains[chain_id] = {
            **config,
            'registered_at': int(time.time()),
            'is_active': True
        }
    
    def add_validator(self, validator_address: str):
        """添加橋接驗證者"""
        if validator_address not in self.validators:
            self.validators.append(validator_address)
            self.threshold = len(self.validators) * 2 // 3 + 1  # 2/3 多數決
    
    def initiate_cross_chain_transfer(self,
                                       source_chain: str,
                                       target_chain: str,
                                       sender: str,
                                       recipient: str,
                                       token: str,
                                       amount: int) -> str:
        """發起跨鏈轉帳"""
        if source_chain not in self.supported_chains:
            raise ValueError(f"不支持源鏈：{source_chain}")
        if target_chain not in self.supported_chains:
            raise ValueError(f"不支持目標鏈：{target_chain}")
        
        tx_id = self._generate_tx_id(source_chain, target_chain, sender, amount)
        
        # 根據橋接類型確定目標代幣
        target_token = self._get_target_token(token, source_chain, target_chain)
        
        tx = CrossChainTransaction(
            tx_id=tx_id,
            bridge_type=self.bridge_type,
            source_chain=source_chain,
            target_chain=target_chain,
            source_token=token,
            target_token=target_token,
            sender=sender,
            recipient=recipient,
            amount=amount
        )
        
        self.pending_txs[tx_id] = tx
        
        # 在源鏈執行鎖定/銷毀操作
        source_tx_hash = self._execute_source_chain_action(tx)
        tx.source_tx_hash = source_tx_hash
        tx.status = TransactionStatus.LOCKED
        
        return tx_id
```

#### 驗證流程

```python
class CrossChainBridgeValidator:
    """驗證者：監聽源鏈事件，驗證並中繼到目標鏈"""
    
    def __init__(self, bridge: CrossChainBridge, private_key: bytes):
        self.bridge = bridge
        self.private_key = private_key
        self.address = self._derive_address(private_key)
    
    def monitor_and_relay(self, source_chain: str):
        """持續監聽源鏈的橋接事件"""
        chain_config = self.bridge.supported_chains[source_chain]
        
        while True:
            events = self._fetch_bridge_events(chain_config['rpc_endpoint'])
            
            for event in events:
                tx_id = event['tx_id']
                if tx_id in self.bridge.pending_txs:
                    self._process_event(event)
    
    def _process_event(self, event: dict):
        """處理橋接事件"""
        tx_id = event['tx_id']
        tx = self.bridge.pending_txs[tx_id]
        
        # 1. 驗證源鏈交易（等待足夠確認數）
        if not self._verify_source_tx(tx, event):
            return
        
        # 2. 生成簽名
        message = self._create_relay_message(tx)
        signature = self._sign_message(message)
        
        # 3. 廣播簽名（通過 P2P 網絡或鏈上）
        self.bridge.submit_validator_signature(tx_id, self.address, signature)
    
    def _verify_source_tx(self, tx: CrossChainTransaction, event: dict) -> bool:
        """驗證源鏈交易的真實性"""
        required_confirmations = {
            'ethereum': 12,
            'bitcoin': 6,
            'polygon': 64,
            'bsc': 15
        }
        
        chain_confirmations = required_confirmations.get(tx.source_chain, 12)
        return event.get('confirmations', 0) >= chain_confirmations
    
    def _create_relay_message(self, tx: CrossChainTransaction) -> bytes:
        """創建中繼消息（由所有驗證者簽名）"""
        data = (
            tx.tx_id.encode() +
            tx.target_chain.encode() +
            tx.recipient.encode() +
            tx.target_token.encode() +
            tx.amount.to_bytes(32, 'big')
        )
        return hashlib.sha256(data).digest()
```

#### 跨鏈驗證（Merkle 證明）

```python
class CrossChainProofVerifier:
    """驗證跨鏈 Merkle 證明"""
    
    def verify_cross_chain_proof(self, tx: CrossChainTransaction) -> bool:
        """
        完整的跨鏈驗證流程：
        1. 驗證 Merkle 包含性證明（交易存在於源鏈區塊中）
        2. 驗證區塊頭（工作量或權益證明）
        3. 驗證交易收據（交易確實被執行）
        """
        if tx.merkle_proof is None:
            return False
        
        proof = tx.merkle_proof
        
        # 步驟 1：驗證 Merkle 路徑（交易哈希 → Merkle Root）
        if not self._verify_merkle_path(
            proof['tx_hash'],
            proof['merkle_path'],
            proof['block_header']['merkle_root']
        ):
            return False
        
        # 步驟 2：驗證區塊頭有效性（PoW/PoS）
        if not self._verify_block_header(proof['block_header'], tx.source_chain):
            return False
        
        # 步驟 3：驗證該區塊是源鏈的一部分（防止孤塊欺騙）
        if not self._verify_chain_ancestry(proof['block_header']['hash'], tx.source_chain):
            return False
        
        # 步驟 4：驗證收據中的事件（確認橋接事件已發出）
        if not self._verify_receipt_logs(proof['receipt'], tx):
            return False
        
        return True
    
    def _verify_merkle_path(self, leaf_hash: str, path: List[dict], expected_root: str) -> bool:
        """驗證 Merkle 路徑"""
        current = leaf_hash
        
        for node in path:
            sibling = node['hash']
            is_left = node['is_left']
            
            if is_left:
                combined = sibling + current
            else:
                combined = current + sibling
            
            current = hashlib.sha256(bytes.fromhex(combined)).hexdigest()
        
        return current == expected_root
    
    def _verify_block_header(self, header: dict, chain: str) -> bool:
        """驗證區塊頭（針對不同共識機制）"""
        if chain == 'bitcoin':
            return self._verify_pow_header(header)
        elif chain in ['ethereum', 'polygon']:
            return self._verify_pos_header(header)
        return False
    
    def _verify_pow_header(self, header: dict) -> bool:
        """驗證 PoW 區塊頭"""
        block_hash = hashlib.sha256(
            hashlib.sha256(self._serialize_header(header)).digest()
        ).hexdigest()
        
        # 區塊哈希必須小於目標難度
        target = int(header['bits'], 16)
        return int(block_hash, 16) < target
    
    def _verify_receipt_logs(self, receipt: dict, tx: CrossChainTransaction) -> bool:
        """驗證收據中包含橋接事件"""
        bridge_event_signature = hashlib.sha256(
            b'BridgeTransfer(address,address,uint256,bytes32)'
        ).hexdigest()
        
        for log in receipt.get('logs', []):
            if (log.get('address') == self.bridge_contract_address and
                log.get('topics', [None])[0] == bridge_event_signature):
                
                # 驗證事件中的金額和地址
                decoded = self._decode_bridge_event(log)
                if (decoded['recipient'] == tx.recipient and
                    decoded['amount'] == tx.amount):
                    return True
        
        return False
```

#### 目標鏈動作執行

```python
class CrossChainExecutor:
    """在目標鏈執行橋接動作"""
    
    def execute_on_target(self, tx: CrossChainTransaction, 
                           validator_signatures: List[dict]) -> str:
        """
        驗證簽名並在目標鏈鑄造/解鎖代幣
        """
        # 1. 驗證簽名數量達到閾值
        if len(validator_signatures) < self.bridge.threshold:
            raise ValueError(f"簽名不足：{len(validator_signatures)}/{self.bridge.threshold}")
        
        # 2. 驗證每個簽名
        relay_message = CrossChainBridgeValidator(None, None)._create_relay_message(tx)
        valid_sigs = 0
        
        for sig_data in validator_signatures:
            if (sig_data['signer'] in self.bridge.validators and
                self._verify_sig(relay_message, sig_data['signature'], sig_data['signer'])):
                valid_sigs += 1
        
        if valid_sigs < self.bridge.threshold:
            raise ValueError("有效簽名不足")
        
        # 3. 防重放檢查（nonce 追蹤）
        nonce_key = f"{tx.source_chain}:{tx.tx_id}"
        if nonce_key in self.bridge.nonce_tracker:
            raise ValueError("交易已處理（重放攻擊防護）")
        self.bridge.nonce_tracker[nonce_key] = int(time.time())
        
        # 4. 在目標鏈執行操作
        if tx.bridge_type == BridgeType.LOCK_MINT:
            target_tx = self._mint_wrapped_token(
                tx.target_chain, tx.target_token, tx.recipient, tx.amount
            )
        elif tx.bridge_type == BridgeType.BURN_MINT:
            target_tx = self._mint_native_token(
                tx.target_chain, tx.target_token, tx.recipient, tx.amount
            )
        else:
            target_tx = self._release_from_pool(
                tx.target_chain, tx.target_token, tx.recipient, tx.amount
            )
        
        tx.target_tx_hash = target_tx
        tx.status = TransactionStatus.COMPLETED
        
        return target_tx
```

#### 橋接類型比較

| 橋接類型 | 信任模型 | 安全性 | 速度 | 代表項目 |
|---------|---------|-------|------|---------|
| 多簽橋 | 外部驗證者 | 中 | 快 | Multichain |
| 最優橋 | 欺詐證明 | 高 | 慢（7天）| Arbitrum Bridge |
| ZK 橋 | 數學證明 | 最高 | 中-快 | zkBridge, Polygon zkEVM |
| HTLC 原子交換 | 無信任 | 最高 | 慢 | 閃電網絡 |
| 流動性網絡 | 流動性提供者 | 中 | 最快 | Hop Protocol |

> **🔄 技術更新（2025+）**：
> - **LayerZero V2（2024）**：採用「去信任」架構，透過獨立的 DVN（去中心化驗證網路）和 Executor 分離設計，應用可自選驗證者組合；已整合超過 70 條鏈，是跨鏈消息傳遞的市場領導者。
> - **Chainlink CCIP（Cross-Chain Interoperability Protocol）**：企業級跨鏈標準，被 SWIFT 等傳統金融機構採用進行跨鏈測試；採用 Risk Management Network 雙重防護。
> - **IBC（Inter-Blockchain Communication，Cosmos）**：Cosmos 生態的原生跨鏈協議，通過輕客戶端驗證實現無需信任的跨鏈通信，已擴展到 Ethereum（IBC Everywhere 計劃）。
> - **Multichain 倒閉事件（2023）**：Multichain 因 CEO 被捕導致橋接癱瘓，超過 1 億美元資產凍結，再次警示多簽橋的中心化風險；促使業界加速轉向 ZK 橋和去信任架構。
> - **ZK Light Client 橋**：Succinct Labs 的 SP1 Helios 等項目實現了用 ZK 證明驗證 Ethereum 的輕客戶端，使任何鏈都能無需信任地驗證 Ethereum 狀態，是跨鏈安全性的最終解決方案。
